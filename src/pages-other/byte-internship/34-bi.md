# 6.13~6.15 自测ing

现在是 6.15 晚上 22:10 分，记录一下这几天的经历吧。

## 6.13~6.14

这两天是周末，依旧是玩了两天，白天就是打游戏 + 看漫画，然后猛吃爽吃，说实话，我知道自己其实应该干点正事的，自测一下，或者学点感兴趣的东西，但是一到周末，就整个一颓废，啥也学不进去，只想放轻松玩一下，后面肯定是要调整一下了，这样下去不是个事啊。

但也搞了一些小玩意，起因是公司的 cursor 额度不够用了，周五晚上委托 jq 姐姐帮忙充了一下 codex，然后就想配置一下使用，本来说好不在 windows 上配置开发环境的，但是这玩意依赖 npm，我想着干脆都要下 npm 了，直接上 msys2 吧，因此就是一套折腾，可以看一下我写的 Agents.md 的片段：

```markdown
- MSYS2 UCRT64 is the primary development environment; prefer `pacman` for toolchains, build tools, Unix-style CLIs, and native libraries.
- Use npm, Go, and `uv` only for their language ecosystem tools; avoid installing the same CLI through multiple managers.
- Use `winget` only as a fallback for Windows-native apps or tools unavailable in UCRT64.
- `C:\Users\18737\Scripts\update-all.ps1` updates in this order: UCRT64 `pacman`, selected npm globals, selected Go tools, then selected `winget` fallback packages.
```

即采用 winget + msys2 作为 win 上的管理器，舒舒服服，非常干净好用啊。

那既然环境配的这么舒服了，那终端也不能差吧，就搞了 oh-my-posh 美化一手，这小亚克力 + 透明底部，谁看了不迷糊，而且还有 oh-my-posh 的插件，自动补全一类的都有，堪比 zsh + oh-my-zsh 的体验了。

那接着回到 codex 上，这不得调教一手，狠狠写了提示词，做了几个小 demo，说实话，这个 codex 很棒，让人很放心，总觉得它写出来的代码我都不用审核就可以直接用，事实上也差不多。

那配好了 codex，怎么能不配一下用了很久的 cc 呢，又是一顿操作，同期我看了大量相关内容，安装了诸如 proxypin、everything 以及一系列很棒的软件，windows 瞬间成为开发机，也是非常之爽啊。

> ps：这次我没有选择常见的 ide 或代码编辑器，而是选择了 zed，无他，又快又开源，还原生继承 ai，没有不用的理由.
